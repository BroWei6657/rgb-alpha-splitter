## 仍未验证

- 电脑 GPU 是否严格输出 RGB 4:4:4 Full 且不做缩放；
- ZCU106 HDMI 2.0 RX 在目标显卡、线缆和 4K60 下的持续锁定；
- AMD HDMI IP 的具体 Vivado 版本和许可；
- ADV7511 双路同步释放后的行/帧偏斜；
- 下游切换台对 HDMI RGB Full/Limited、KEY 灰阶和时序的解释；
- EMC、ESD、温升、24 小时稳定性和不同线缆互操作；
- 真实端到端延迟。

## 有硬件后的通过标准

| 类别 | 通过标准 |
| --- | --- |
| 输入锁定 | 4K60 连续 24 小时，无 RX unlock |
| 帧完整性 | 测试 ramp 运行 10^9 帧，重复行和 KEY RGB 错误计数为 0 |
| 输出同步 | 两路 VS 偏斜 <= 1 个 148.5 MHz 像素时钟；HS 无累计漂移 |
| 输出帧率 | 两路均为 60/1，10^6 帧无重复/丢失 |
| KEY 码值 | 0/16/128/235/255 误差 <= 1 LSB，范围配置明确 |
| FILL 色彩 | Rec.709 灰阶和色条误差满足目标切换台的 8-bit HDMI 容差 |
| 故障模式 | 拔掉输入后 2 帧内两路同时黑；恢复 30 帧稳定后同时恢复 |
| 端到端延迟 | 目标 < 1 ms；若超过必须说明引入的帧缓存 |

## 复现命令

```powershell
node overlay-link-lab\tests\transport-core.test.js
node overlay-link-lab\scripts\verify-theory.js
```

浏览器模拟器的错误注入开关必须同时使“重复行一致性”和“源数据一致性”变为 FAIL；关闭后两项恢复 PASS。
